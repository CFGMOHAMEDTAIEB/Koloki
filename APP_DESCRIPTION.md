---

## 🎯 WHAT IS KOLOKI?

**Koloki** is a modern, secure colocation marketplace platform that connects **house owners** with **renters** looking for shared accommodation. It's a comprehensive solution built with enterprise-grade security, transparent pricing, built-in payment processing, and multi-language support.

**Key Tagline**: "Find your perfect home share. Safe. Transparent. Easy."

**Target Market**: 
- Tunisia & MENA region
- Young professionals
- Students
- Expats
- Budget-conscious renters

---

## 🌍 PLATFORM OVERVIEW

### The Problem It Solves
1. **Trust Gap** - Renters don't know if properties are safe
2. **Transparency** - Unclear pricing and hidden fees
3. **Communication** - No built-in messaging system
4. **Payment Safety** - No secure, centralized payment method
5. **Verification** - No way to verify users' identities

### The Solution
Koloki provides:
- ✅ Verified users and properties
- ✅ Transparent commission structure
- ✅ Secure payment processing
- ✅ Direct messaging system
- ✅ Transaction history tracking
- ✅ Mobile-responsive interface
- ✅ Multi-language support

## 💰 BUSINESS MODEL & PRICING

### Revenue Streams

#### 1. **Listing Fees** 💵
- **Who Pays**: House owners
- **Amount**: 10 DT per property listing
- **When**: When creating a property listing
- **Purpose**: To maintain property quality

#### 2. **Commission** 📊
- **Who Pays**: House owners
- **Amount**: 10% of booking price
- **When**: When booking is confirmed
- **Purpose**: Platform maintenance & features

**Example**: 
```
Rent: 500 DT/month
Commission (10%): 50 DT → Koloki
Owner receives: 450 DT
```

#### 3. **Advertising Revenue** 📢
- **Who Pays**: Advertisers
- **Amount**: CPM-based (e.g., $1 per 1000 impressions)
- **When**: Ads viewed or clicked
- **Purpose**: Additional revenue stream

#### 4. **Premium Features** (Future) 🚀
- Featured listings
- Priority customer support
- Advertisement for owners' properties
- Advanced analytics

## 📊 BACKEND API

### API Architecture
- **Framework**: Express.js (Node.js)
- **Language**: TypeScript (fully typed)
- **Database**: MongoDB (NoSQL)
- **Authentication**: JWT tokens
- **Payment**: Stripe integration

### Core API Endpoints

#### **Authentication Routes**
```
POST   /api/auth/register      - Register new user
POST   /api/auth/login         - User login
POST   /api/auth/logout        - User logout
Post   /api/auth/refresh       - Refresh JWT token
GET    /api/auth/me            - Get current user
```

#### **Property Routes**
```
GET    /api/properties         - List all properties
POST   /api/properties         - Create new listing
GET    /api/properties/:id     - Get property details
PUT    /api/properties/:id     - Update property
DELETE /api/properties/:id     - Delete property
GET    /api/properties/search  - Search with filters
```

#### **Message Routes** (NEW!)
```
GET    /api/messages/conversations        - List conversations
GET    /api/messages/conversations/:id    - Get conversation
POST   /api/messages/conversations        - Create conversation
POST   /api/messages                      - Send message
PATCH  /api/messages/:id/read             - Mark as read
GET    /api/messages/unread-count        - Get unread count
GET    /api/messages/search               - Search conversations
```

#### **Payment Routes**
```
POST   /api/payments/calculate   - Calculate with commission
POST   /api/payments/process     - Process payment
GET    /api/payments/methods     - Available methods
POST   /api/payments/refund      - Process refund
GET    /api/payments/history     - Transaction history
```

#### **User Routes**
```
GET    /api/users/:id           - Get user profile
PUT    /api/users/:id           - Update profile
GET    /api/users/:id/properties - User's listings
```

#### **Verification Routes** (Admin)
```
GET    /api/verification/pending        - Pending users
POST   /api/verification/:id/approve    - Approve user
POST   /api/verification/:id/reject     - Reject user
```

#### **Advertising Routes**
```
GET    /api/advertising            - List ads
POST   /api/advertising            - Create ad
POST   /api/advertising/:id/click  - Track click
POST   /api/advertising/:id/impression - Track view
```
